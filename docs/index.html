Governed workflow

From research question to auditable result.

Evidence Bench separates generation from acceptance. “Supported” means the configured evidence gates passed—not peer review, clinical authority, science lock, or submission readiness. Consequential scientific and medical decisions still require human review.

  1. 01
    ⌁

    Lock the method

    Upload files and define the objective. Plan linting and an independent audit must pass before analysis begins.

  2. 02
    λ

    Build the evidence

    Qwen retrieves sources and runs confined Python and R. Every successful artifact is hashed and retained.

  3. 03
    ✓

    Test every claim

    Controller code checks statistical consistency, source bindings, reconciliation, figures, tables, and requested outputs.

  4. 04
    ◉

    Audit independently

    Gemma reviews the argument and actual rendered displays. Actionable findings return for bounded correction.

A scientific workbench, not a chat window

Everything needed to inspect how a result was made.

Use the browser for hands-on work or connect existing agents through REST, A2A, the CLI, or the bundled Codex and Claude skill.

01

Isolated scientific computing

Python and R receive source files read-only and write only to bounded output directories. Reader-facing figures default to R with a preinstalled publication, tidy-data, Excel, and statistical stack; specialist packages remain isolated and version-recorded.

◇

Explicit trust boundaries

Scientific files are not ordinary chat attachments.

Evidence Bench treats uploads, generated code, websites, browser downloads, and model output as untrusted inputs. Isolation and acceptance rules live in controller code and separate services.

  • ✓Mutually isolated workspaces
  • ✓Read-only analysis inputs
  • ✓Offline Python and R workers
  • ✓Internal-only browser DevTools
  • ✓Private-address-denying browser proxy
  • ✓No chain-of-thought in provenance
